
# 🧹 Инструкция по настройке конфигурации для очистки Nexus-репозиториев

Файл конфигурации (`.yaml`) описывает правила, по которым скрипт будет определять, какие образы можно удалить, а какие — нужно сохранить.

Пример файла:

```yaml
repo_names:
  - test1

regex_rules:
  "^dev-":
    retention_days: 5
    reserved: 2
  "^release-.*":
    retention_days: 15
    reserved: 3

# Для всех, кто не попал под regex_rules
no_match_retention_days: 10
no_match_reserved: 1

# Глобальное ограничение возраста (в днях), применяется ко всем
max_retention_days: 30

# true — ничего не будет удалено, только логгирование
dry_run: true
```

## 📘 Описание параметров

| Поле | Описание |
|------|----------|
| `repo_names` | Список репозиториев, в которых будет производиться очистка |
| `regex_rules` | Словарь с шаблонами версий (регулярные выражения). Для каждой группы можно задать `retention_days` (срок хранения) и `reserved` (количество последних, которые нельзя удалять) |
| `no_match_retention_days` | Срок хранения по умолчанию, если версия не подходит ни под одно правило |
| `no_match_reserved` | Кол-во последних артефактов без совпадений, которые нужно оставить |
| `max_retention_days` | Абсолютный лимит: если артефакт старше этого срока — он удаляется, даже если попадает под `reserved` или `retention_days` |
| `dry_run` | Если `true`, скрипт **не удаляет**, а только пишет в лог, что бы удалил |

## 📊 Поведение при разных комбинациях настроек

| `retention_days` | `reserved` | `max_retention_days` | Что будет? |
|------------------|------------|-----------------------|------------|
| ❌ (не указан)   | ❌          | ❌                    | Компонент сохраняется |
| ✅               | ❌          | ❌                    | Удаляется, если компонент старше `retention_days` |
| ❌               | ✅          | ❌                    | Сохраняются последние `reserved`, остальные удаляются |
| ✅               | ✅          | ❌                    | Сначала учитывается `reserved`, потом `retention_days` |
| ✅/❌            | ✅/❌       | ✅                    | Если компонент старше `max_retention_days` — **удаляется в любом случае** |
| ❌               | ❌          | ✅                    | Удаляется, если возраст превышает `max_retention_days` |

## 🧪 Пример логики

Допустим, у вас есть 6 образов:

```
release-1.0 (45 дней назад)
release-1.1 (20 дней назад)
release-1.2 (10 дней назад)
release-1.3 (5 дней назад)
release-1.4 (2 дня назад)
release-1.5 (1 день назад)
```

При настройках:
```yaml
regex_rules:
  "^release-":
    retention_days: 15
    reserved: 3
max_retention_days: 30
```

Результат:
- `release-1.0` — удаляется (старше `max_retention_days`)
- `release-1.1` — удаляется (не входит в 3 последних, и старше `retention_days`)
- `release-1.2`, `1.3`, `1.4` — сохраняются (входят в `reserved`)
- `release-1.5` — сохраняется

## 🔒 Особенности

- Тег `latest` **никогда не удаляется**
- Артефакты без `version`/`name`/`lastModified` — игнорируются
- Все действия логгируются в файл `logs/cleaner.log`
- Если `dry_run: true`, ничего не будет удалено, только выведено в лог
