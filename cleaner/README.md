# 🧹 Инструкция по настройке конфигурации для очистки Nexus-репозиториев

Файл конфигурации (`.yaml`) описывает правила, по которым скрипт будет определять, какие образы можно удалить, а какие — нужно сохранить.

## 📁 Пример файла конфигурации

```yaml
repo_names:
  - test1

regex_rules:
  "^dev-":
    retention_days: 5
    reserved: 2
  "^release-.*":
    retention_days: 15
    reserved: 3
    min_days_since_last_download: 7

no_match_retention_days: 10
no_match_reserved: 1
no_match_min_days_since_last_download: 21

max_retention_days: 30
dry_run: true
```

---

## 📘 Описание параметров

| Поле                                    | Описание                                                                |
| --------------------------------------- | ----------------------------------------------------------------------- |
| `repo_names`                            | Список репозиториев, в которых будет производиться очистка              |
| `regex_rules`                           | Словарь с шаблонами версий (регулярные выражения)                       |
| `retention_days`                        | Срок хранения для совпадающих по regex компонентов                      |
| `reserved`                              | Количество последних компонентов, которые нельзя удалять                |
| `min_days_since_last_download`          | Минимальное число дней с последнего скачивания                          |
| `no_match_retention_days`               | Срок хранения, если не совпадает ни с одним regex                       |
| `no_match_reserved`                     | Количество последних компонентов без совпадений, которые нужно оставить |
| `no_match_min_days_since_last_download` | Минимальные дни с последнего скачивания без совпадений                  |
| `max_retention_days`                    | Абсолютный лимит: старше этого срока — удаляется                        |
| `dry_run`                               | `true` — только логирование, без удаления                               |

---

## 🧮 Приоритет применения правил

| Приоритет | Параметр                                                                 | Где применяется            | Условие применения                                          |
| --------- | ------------------------------------------------------------------------ | -------------------------- | ----------------------------------------------------------- |
| 1         | `max_retention_days`                                                     | Все компоненты             | **Безусловно удаляется**, если старше указанного срока      |
| 2         | `reserved` / `no_match_reserved`                                         | Внутри правила / глобально | **Сохраняется** N самых новых компонентов                   |
| 3         | `min_days_since_last_download` / `no_match_min_days_since_last_download` | Внутри правила / глобально | **Не удаляется**, если скачан менее X дней назад            |
| 3         | `retention_days` / `no_match_retention_days`                             | Внутри правила / глобально | Удаляется, если старше срока и не защищён другими условиями |

> ❗ Тег `latest` **никогда не удаляется**

## 🧪 Пример работы

### 📦 Образы:

```text
release-1.0 (45 дней назад)
release-1.1 (20 дней назад)
release-1.2 (10 дней назад)
release-1.3 (5 дней назад)
release-1.4 (2 дня назад)
release-1.5 (1 день назад)
```

### 🔧 Конфигурация:

```yaml
regex_rules:
  "^release-":
    retention_days: 15
    reserved: 3
max_retention_days: 30
```

### 📋 Результат:

| Образ       | Возраст (дней) | В reserved? | Старше max? | Старше retention? | Итог                             |
| ----------- | -------------- | ----------- | ----------- | ----------------- | -------------------------------- |
| release-1.0 | 45             | ❌           | ✅           | ✅                 | ❌ Удаляется (max_retention_days) |
| release-1.1 | 20             | ❌           | ❌           | ✅                 | ❌ Удаляется (retention_days)     |
| release-1.2 | 10             | ✅           | ❌           | ❌                 | ✅ Сохраняется (reserved)         |
| release-1.3 | 5              | ✅           | ❌           | ❌                 | ✅ Сохраняется (reserved)         |
| release-1.4 | 2              | ✅           | ❌           | ❌                 | ✅ Сохраняется (reserved)         |
| release-1.5 | 1              | ❌           | ❌           | ❌                 | ✅ Сохраняется (молодой)          |

## ✅ Выводы

- `max_retention_days` — самый строгий критерий удаления.
- `reserved` — защищает последние образы.
- `retention_days` — удаляет старые, если они вне `reserved`.
- `min_days_since_last_download` — защищает активные образы.
- Всё, что не подпадает под правила — регулируется `no_match_*` параметрами.
