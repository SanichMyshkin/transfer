# 🧹 Nexus Docker Cleaner

Скрипт для автоматической очистки Docker-репозиториев в **Nexus 3** по правилам хранения образов: сроку жизни (retention) и количеству резервных версий (reserved).

---

## 📌 Описание

Этот скрипт подключается к REST API Nexus 3 и удаляет устаревшие компоненты (образы), основываясь на следующих параметрах:

---

## ⚙️ Настройки

Перед запуском необходимо задать переменные окружения (можно через `.env`):

```env
USER_NAME=your-nexus-username
PASSWORD=your-nexus-password
BASE_URL=https://your-nexus-domain/
DRY_RUN=true   # true — ничего не удаляет, только логирует
```
- 🟢 retention (в днях) — сколько времени можно хранить образы
Это ограничение по максимальному возрасту.
Если образ старше retention, и он не в числе "зарезервированных", то он удаляется.
>  Пример: если retention 8 дней, то всё старше этого срока может быть удалено.

- 🔵 2. reserved (по количеству) — сколько самых свежих версий нужно обязательно сохранить
Это ограничение по количеству последних версий.
Даже если они старше срока, их нельзя удалить, если они входят в резерв.
> Пример: если reserved = 1, то самая свежая версия с данным префиксом и именем никогда не удалится, даже если ей 100 дней.
